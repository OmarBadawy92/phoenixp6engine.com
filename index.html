What's Inside

THREE POWERFUL MODULES

Phoenix P6 Engine is organized into 3 tabs — each targeting a critical area of your P6 workflow.

⚡

XER Import

Ultrafast import of Primavera P6 XER files directly into SQLite — no Oracle, no P6 server, no admin needed. Handles large schedules in seconds.

🔄

DB → DB Copy

Copy an entire project from one SQLite database to another. All relationships, codes, calendars, and resources preserved perfectly.

📋

Duplicate Projects

Instantly clone a project within the same database. Perfect for creating scenario copies or baseline variations without re-importing.

🔗

Assign / Un-assign Baseline

Link or unlink baseline projects with full integrity checks — cleans up dangling references automatically. Safe and reversible.

🔍

Out-of-Sequence Detector & Solver

Automatically find all out-of-sequence activities in your schedule and apply fixes — works on XER and SQLite databases.

✂

Split Activities

Split activities to create revised schedules — properly handles actual start, dates, durations, and task types. Works on both XER and SQLite.

👤

Remove Resource Assignments

Bulk-remove resource assignments from selected projects. Clean up your schedule before re-assignment or for submittal-ready exports.

🔒

Delete Shared Mode

Remove the shared mode lock from projects stuck in shared/exclusive mode — no need for P6 client access or DBA privileges.

🔣

Activity ID Limit Fix

Resets P6's internal NEXTKEY counter to allow new activity IDs beyond the default limit — a common pain point for large projects.

🧹

Clean Rubbish Data

Detect and remove orphaned records, dangling foreign keys, and corrupted data entries that slow down or break P6 databases.

📄

Copy Database Layouts

Transfer Layouts, Filters, and Units of Measure between databases — saves hours of manual recreation after migrations.

⏪

Rollback Database Version

Downgrade a SQLite database schema to an earlier P6 version — critical when sharing databases with teams using older P6 versions.

🔐

Reset Password

Reset user passwords directly in the SQLite database — no need for P6 admin access or DBA support. Fast and secure.
